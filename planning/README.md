# Planning

Skills for shaping work before code gets written — interviewing yourself, writing requirements, slicing scope, and promoting deferred ideas into real workstreams.

| Skill | Purpose |
|-------|---------|
| [`grill-me`](./grill-me) | Stress-test a plan or design by interviewing you relentlessly until every branch of the decision tree is resolved. |
| [`write-a-prd`](./write-a-prd) | Produce a PRD through user interview, codebase exploration, and module design, then file it as a GitHub issue. |
| [`prd-to-issues`](./prd-to-issues) | Break a PRD into independently-grabbable GitHub issues using tracer-bullet vertical slices. |
| [`promote-feature`](./promote-feature) | Turn a `docs/deferred/` stub into a ralph-loop-ready scope: move the doc, file issues with a `feature/<slug>` label, initialize `plans/`. |
